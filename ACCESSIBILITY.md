# 可访问性指南 (Web Content Accessibility Guidelines - WCAG Implementation)

GlobalMart 致力于为所有用户提供无障碍的购物体验。本项目在 **Block W (Accessibility)** 的实现中，严格参考了国际主流的 Web 内容可访问性指南，并结合中国本土市场需求进行了深度定制。

## 1. 核心可访问性特性 (Accessibility Features)

### A. 长辈模式 (Senior Mode)
*   **一键切换：** 在导航栏顶部提供了醒目的“长辈模式/标准模式”切换按钮。
*   **字号放大：** 开启后，全站基础字号提升至 1.25rem (20px)，标题字号放大至 2rem 以上，确保视力不佳的用户能清晰阅读。
*   **布局简化：** 移除了复杂的装饰性间距，简化了 Brutalist UI 的重叠阴影，改为更直观的高对比度边框。
*   **交互增强：** 按钮和输入框的点击区域增大，行高增加至 1.6，防止误触并提升阅读舒适度。

### B. 本土化与国际化 (Localization & i18n)
*   **多语言支持：** 系统完整支持中英双语切换，所有 UI 文本均经过专业翻译。
*   **计价单位本土化：** 
    *   **中文模式：** 计价单位自动切换为人民币 (￥)，并应用 7.2 的 demo 汇率转换。
    *   **英文模式：** 计价单位显示为美元 ($)。
    *   **精度适配：** 在长辈模式下，价格自动取整（不显示分位），减少视觉干扰。

### C. 多元化支付辅助 (Inclusive Payment)
*   **支付方式 demo：** 在结算页面，系统会根据当前语言环境动态调整支付优先级：
    *   **中文环境下：** 优先展示中国用户习惯的 **微信支付 (WeChat Pay)** 和 **支付宝 (Alipay)**，并提供模拟扫码确认流程。
    *   **英文环境下：** 默认选中 **国际信用卡 (Credit Card)** 支付。
*   **无障碍表单：** 支付确认模态框使用了高对比度设计，并配备了清晰的取消与确认逻辑。

## 2. 技术合规性 (Technical Compliance)

### D. ARIA 语义化与屏幕阅读器支持
*   **Role 属性：** 关键组件（如 `Cart`, `Checkout`, `Shop`）均添加了 `role="main"` 和 `aria-label`。
*   **状态反馈：** 所有的加载状态（Loading）均使用了带有文字说明的动画，并配置了 `aria-live` 属性（通过语义化标签体现）。
*   **表单验证：** 所有输入框均有对应的 `label` 和 `placeholder`，并支持键盘 Tab 键导航。

### E. 视觉设计原则
*   **高对比度：** 采用 Brutalism (新野兽主义) 设计风格，使用纯黑 (#000000) 边框与高饱和度背景色，确保文字与背景的对比度远超 WCAG AA 级标准。
*   **响应式适配：** 适配从移动端到超大桌面端的各种屏幕尺寸，确保在缩放状态下功能不丢失。
